# Saymore

Saymore 将用户的自然语音实时识别为文字，并把结果可靠地投递到当前输入位置。

## Language

**口述会话**：
从用户开始录音到文本成功投递、取消或失败的完整生命周期。
_Avoid_: 录音任务、听写任务

**口述上限**：
一次口述允许持续的最长时间；达到上限会自动结束录音并继续处理已有内容，而不是取消会话。
_Avoid_: 超时、配额

**取消**：
用户在投递前主动放弃整次口述；系统终止当前处理，清除会话内音频和文本，并且不投递、不写入本地历史。
_Avoid_: 失败、超时、达到上限

**切换式口述**：
用户完整单击一次全局快捷键开始口述，再完整单击一次结束口述的默认录音交互；取消使用独立命令。
_Avoid_: 按住说、松开结束

**原始转录**：
语音识别器根据音频产生、尚未经过格式化和词典校正的文字。
_Avoid_: 原文、识别稿

**最终转录**：
语音识别完成后得到的稳定文字，是后续可选变换阶段的输入。
_Avoid_: 精炼文本、AI 文案

**规则标准化**：
在可选 LLM 精炼之后执行的确定性变换。当前只把已存在于个人词典中的 ASCII 字母数字标准词统一为用户保存的大小写和全角/半角形式，例如 `openai`、`OPENAI` 和 `ＯｐｅｎＡＩ` 统一为 `OpenAI`；不把 `open ai` 等不同词序列推断成同一词，也不使用错误写法做语义映射。
_Avoid_: 精炼、润色

**精炼文本**：
LLM 在不改变事实、观点和表达目的的前提下，对安全本地清理后的最终转录去除口语冗余、处理自我修正并整理结构后得到的文字。规则标准化在它之后运行。
_Avoid_: 回答、续写、AI 文案

**精炼模型**：
被限制为只处理最终转录的 LLM Provider；模型能力可以是通用的，但产品不向它开放问答、屏幕上下文或任务执行。
_Avoid_: 语音助手、聊天机器人

**投递**：
将精炼文本或降级后的最终转录写入实际投递时光标所在的位置，不包含自动发送消息。
_Avoid_: 发送、粘贴

**投递目标**：
实际投递发生时获得焦点的可编辑控件及其光标或选区；它不绑定口述开始时的应用或输入框。存在选区时最终文本替换选区，但 Saymore 不读取或处理选区原文。
_Avoid_: 原始输入框、录音目标

**Secure 输入控件**：
由操作系统标记为密码或其他安全文本输入的控件。Saymore 可以向实际投递时获得焦点的 Secure 输入控件尝试受限投递，但不读取或验证其中的内容，且该次口述不写入本地历史。
_Avoid_: 普通不可编辑控件、投递失败目标

**受限投递**：
向无法安全观察内容或选区的投递目标发起的一次不可验证投递；它只表示系统已发出输入操作，不等同于确认文字已经写入，也不因无法验证而自动判定为失败。
_Avoid_: 投递成功、验证投递

**最终投递**：
口述结束且所有已启用的变换阶段完成后，将一个最终纯文本结果一次性写入目标输入位置。结果浮层和历史复制使用同一文本表示，不生成 HTML 或 RTF。
_Avoid_: 实时投递、流式输入

**临时剪贴板投递**：
macOS 直接 Accessibility 写入明确失败或目标控件不提供可验证选区范围时的降级路径。Saymore 只在单次操作内存中保存当前系统剪贴板、写入最终纯文本、触发粘贴，并在用户没有同时修改剪贴板时恢复原内容。该恢复快照在操作后立即释放，不进入本地历史；第三方剪贴板管理器自己的历史不受 Saymore 控制。
_Avoid_: 本地历史、结果浮层复制、永久剪贴板记录

**切换式口述**：
完整单击一次全局触发器开始录音，再完整单击一次结束并处理；首次安装只有一个主触发器，macOS 当前默认为右 `Command`。主触发器可在输入框中直接录入替换，“添加另一个”用于添加并行的键盘或鼠标触发器。左右主按键和滚轮滚动不可绑定。Saymore 可录音时消费已绑定鼠标事件，不可录音时恢复透传。录音期间再次触发任一已绑定触发器都会结束同一会话。
_Avoid_: 按住模式、系统听写

**应用行为设置**：
登录时启动和 Dock 图标显示两个独立开关，默认都开启。关闭 Dock 图标后菜单栏入口仍然存在；关闭设置或历史窗口不退出应用，只有菜单栏退出或 `Cmd+Q` 才终止进程。
_Avoid_: 常驻状态、单一入口模式

**活动会话退出**：
录音中退出会取消口述并立即终止进程；处理阶段退出时由用户选择等待完成或立即丢弃。等待 30 秒后只提供继续等待或明确丢弃，不自动删除内容。等待完成后不直接投递：历史开启则保存结果后退出，历史关闭则在结果浮层中复制或明确放弃后退出。
_Avoid_: 关闭窗口、暂停 Saymore

**暂停 Saymore**：
保留应用进程、菜单栏和设置入口，但禁用所有录音触发器并恢复鼠标事件透传。录音中暂停等同取消当前口述。暂停状态跨应用重启和系统登录保留，只有用户从菜单栏明确恢复才重新启用触发器。
_Avoid_: 退出应用、暂停录音

**音频设置**：
麦克风默认自动检测，即每次录音开始时使用 macOS 当前默认输入设备；也可由用户固定设备。录音过程中设备断开时立即结束，保留已采集音频继续识别，本次会话不切换设备。交互声音与“语音输入时静音其他活动音频”分别默认开启并可关闭。交互声音只用于录音开始和成功投递结束；主动取消、录音失败、识别失败和投递失败只使用视觉反馈。开始提示音播放后再静音；录音一结束就恢复音量，ASR 与精炼期间不继续静音。若用户在录音期间主动调节音量，则保留用户的新设置而不恢复旧值。
_Avoid_: ASR 音频参数、系统声音设置

**空会话**：
VAD 没有检测到有效语音的录音。它正常结束，但不调用 ASR 或 LLM、不投递、不写历史、不显示结果浮层，只给出短暂的“未检测到语音”反馈。
_Avoid_: 取消、识别失败

**渐进权限引导**：
首次使用时先申请麦克风权限，通过后由用户确认并开始下载推荐默认模型；下载与辅助功能授权、触发器设置并行。模型校验完成后，引导用户在任意外部可编辑输入框运行正式管线完成第一次真实听写，不提供专用测试框或自动控制 TextEdit。第一次听写可跳过，菜单栏保留“完成设置”；缺失条件只禁用相关能力。权限拒绝后只提供系统设置入口和重新检测，不循环弹窗；不申请屏幕录制、通讯录或完整磁盘访问。
_Avoid_: 一次性权限墙、后台权限申请

**结果浮层**：
明确找不到投递目标或投递失败时展示只读最终文本并允许用户复制的恢复界面；仅仅无法验证受限投递不自动展示，避免文字已经写入后再次打断用户。结果浮层保持到复制或主动关闭，并在本地历史启用时于新口述开始前保存未处理结果。
_Avoid_: 预览浮层、错误弹窗

**仅结果浮层模式**：
辅助功能权限未授予或被撤销时的降级模式。录音、识别和精炼仍可运行，但不调用直接投递接口，最终文本始终进入结果浮层；授权恢复后下一次口述重新启用直接投递。
_Avoid_: 投递失败、剪贴板模式

**状态浮条**：
正常口述时显示录音、处理、成功或失败状态及音量反馈的轻量界面；它不展示转录文本，并可由用户关闭。
_Avoid_: 结果浮层、文本预览

**本地历史**：
保存在用户设备上的最终输出记录；它可以包含时间、投递状态和启用阶段，但不包含音频。普通发布只保存最终交付结果；调试构建或显式启用 `history-experiments` 的测试构建还可在同一加密记录中保存 ASR 原始结果和成功通过检查的 LLM 精炼结果，用于三阶段对比。首版只支持查看、复制、单条删除、全部清空和按期限自动清理，不承担搜索、标签、收藏或同步职责。
_Avoid_: 录音历史、调试日志

**个人词典**：
用户在应用内手动添加、通过 CSV 导入，或未来由高可信纠正信号自动加入的正式词条集合。当前运行版本只支持前两种入口；自动加入仍是长期核心能力，但暂时停用。只有与本次转录相关的条目才进入后续识别、精炼或规则标准化。
_Avoid_: 知识库、记忆、候选审核队列

**词汇观察**：
为未来自动加入词典保留的最小纠正信号。现有数据库表和兼容数据继续保留，但当前运行链路不再根据 ASR 与 LLM 或最终文本的差异生成观察，因为模型润色差异不等于用户纠正。待产品取得高可信用户纠正信号后再重新设计采集、置信度和保留策略。
_Avoid_: 原始转录历史、完整上下文、待确认候选

**自动词典学习**：
Saymore 的长期核心能力：根据高可信用户纠正信号自动加入标准词，并提供可理解、可撤销的反馈。当前基于 ASR 与 LLM 差异的实现及其浮窗暂时停用，不把模型改写误当作用户意图；这不代表取消自动词典学习。
_Avoid_: ASR/LLM 差异猜测、未经验证的语义映射

**词典静默期**：
为未来自动加入恢复后预留的内部抑制策略。现有表和兼容数据继续保留，当前运行版本不创建新的静默记录；具体期限和恢复规则须与高可信纠正信号方案一并重新确认，不作为复杂 UI 状态展示。
_Avoid_: 禁用词条、永久屏蔽

**词典条目**：
正式的标准写法、可选语言标签、来源和时间。旧的常见错误写法字段为数据兼容和未来高可信学习保留，当前不由 UI 或 CSV 创建，也不参与本地语义映射。MVP 中词条全局生效并按语言隔离，不区分目标应用；当前规则标准化只统一同一 ASCII 字母数字词的大小写和全角/半角形式，并避开 URL、邮箱、路径、域名和下划线标识符。
_Avoid_: 词汇观察、知识条目

**词典文件**：
CSV 是当前唯一的词典文件入口，用于从其他产品迁移正式词条；每行包含标准写法和可选语言标签。当前不提供 Markdown、JSON 或其他导出，也不把外部文件作为第二个数据源。
_Avoid_: 完整应用备份、实时同步文件、Markdown 投影、导出

**本地数据密钥**：
用于 AES-256-GCM 加密本地听写历史的随机 32-byte 密钥，保存在 macOS Keychain 或 Windows Credential Manager。它与 Provider API Key 分离；密钥不可用时停止访问旧历史，用户可显式删除旧历史并重新初始化，不得静默改用明文。个人词典和词汇观察不使用该密钥，也不受历史重置影响。
_Avoid_: API Key、用户密码

**Provider 数据确认**：
云端 Provider 首次启用前的数据发送确认，明确发送内容、目标服务地址，以及是否包含本次相关的正式词典条目。更换 Provider、Base URL 或数据范围后重新确认；本地 Provider 不需要。词汇观察永远不发送。
_Avoid_: 通用隐私政策确认、每次口述确认

**可选遥测**：
默认关闭、由用户分别主动开启的匿名使用统计和崩溃报告。事件采用字段白名单，禁止包含音频、任何文本内容、历史、词典、候选、Provider 地址、凭据或 Provider 请求响应。
_Avoid_: 默认诊断、听写质量数据上传

**应用更新检查**：
默认开启的签名版本查询，不等于静默更新，也不是遥测。发现新版本后由用户确认，社区构建也可直接下载并安装经过 Saymore 更新签名验证的产物；请求不携带稳定设备标识或用户数据。
_Avoid_: 静默更新、更新遥测

**Saymore 更新签名**：
独立于 Apple Developer ID 的项目发布签名。客户端用内置公钥验证更新清单和安装包，签名无效时拒绝安装；私钥只存在于受保护的发布环境。它保证更新来自 Saymore，但不能替代 Gatekeeper 对 Apple 开发者身份的信任。
_Avoid_: Apple 代码签名、文件哈希

**社区构建**：
当前没有 Apple Developer Program 账号时发布的未公证 macOS 构建。它使用 Saymore 自有发布签名和哈希验证产物完整性，但需要用户按说明通过 Gatekeeper 手动放行，不能称为 Apple 签名或公证构建。
_Avoid_: 正式 macOS 构建、已签名应用

**正式 macOS 构建**：
取得 Apple Developer Program 账号后，使用 Developer ID 代码签名并通过 Apple Notarization、面向普通用户分发的安装包。
_Avoid_: 社区构建、源码构建

**可信发布源**：
MVP 中唯一发布真相是 GitHub Releases；官网只链接具体 Release。一个完整发布包含 DMG 安装包、应用内更新归档、更新清单、Saymore 更新签名、SHA-256 和发行说明。
_Avoid_: 官网镜像、第三方软件源

**推荐默认模型**：
某个 Saymore 应用版本为新安装推荐的独立模型版本。应用与模型分别版本化，应用同时声明兼容模型范围；已有模型仍兼容时不因应用升级被强制替换。
_Avoid_: 内置模型、与应用绑定的模型

**Saymore 验证模型**：
第三方或自有模型中，被 Saymore 的签名适配清单固定上游版本、许可证、下载地址、哈希、兼容范围和测试结果的模型。默认模型权重必须允许个人与商业本地推理且不限制合法用途，运行时代码采用 OSI 批准许可证；权重允许再分发不是硬要求。默认模型不能要求账号注册、个人信息或额外条款接受。清单签名证明适配元数据来自 Saymore，不表示 Saymore 拥有或创作该模型。
_Avoid_: Saymore 官方模型、自研模型

**签名适配清单**：
由 Saymore 签名的模型兼容性与供应链元数据。客户端验证清单签名以及模型文件哈希和大小；验证失败时拒绝安装。清单优先指向上游不可变版本，可列出产物哈希完全一致的镜像；镜像不改变模型来源或许可证。
_Avoid_: 模型签名、模型版权证明

**模型切换保护**：
新模型启用时保留旧模型，直到新模型至少成功完成一次真实识别。新模型加载或识别失败时自动回退；旧模型只在用户确认后删除。
_Avoid_: 模型自动清理、无回退切换

**ASR Model Hub**：
MVP 后的中文优先模型发现与管理能力。它基于签名适配清单展示兼容的本地模型和云端 ASR Provider，并提供按需安装、能力比较、硬件基准与后续社区排名；它不改变 Saymore 首先是语音输入工具，也不属于首版范围。
_Avoid_: MVP 模型设置、任意模型执行器、权重托管服务

**社区模型**：
由社区提交元数据或测试结果、但未完成 Saymore 全量验收的模型变体。社区条目只能引用当前应用已包含并信任的 Runtime Adapter；不得通过模型清单下载并执行任意代码。未达到验证门槛时必须与 Saymore 验证模型分开展示。
_Avoid_: Saymore 验证模型、手动导入的未知模型

**模型排名**：
MVP 后按模型精确版本、量化变体、运行时、设备和测试集生成的可审计比较。客观 benchmark、兼容性、安装趋势、活跃使用和社区评价分别展示，不合成不透明总分；统计默认关闭且禁止上传音频或文本。
_Avoid_: 下载量排行、跨设备总榜、厂商宣传榜

**识别能力**：
某个 ASR Provider 明确声明并经过验证的语言、自动检测、句内混说、地区变体、热词、流式和离线能力集合。
_Avoid_: 产品语言列表、模型宣传能力

**语言策略**：
一次口述采用的语言选择方式，可以是自动检测、带偏好的自动检测或固定语言；可选策略受当前识别能力约束。
_Avoid_: 界面语言、翻译模式

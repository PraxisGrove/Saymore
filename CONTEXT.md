# Saymore

Saymore 将用户的自然语音实时识别为文字，并把结果可靠地投递到当前输入位置。

## Language

**口述会话**：从用户开始录音到文本成功投递、取消或失败的完整生命周期。_Avoid_:
录音任务、听写任务

**口述会话标识**：
一次口述会话从开始到结束保持不变的唯一身份；正常结束、取消后的撤销恢复、历史记录、词汇观察和诊断事件都使用同一身份。
_Avoid_: 历史记录标识、词汇观察标识、处理批次标识

**口述上限**：
一次口述允许持续的最长时间；达到上限会自动结束录音并继续处理已有内容，而不是取消会话。
_Avoid_: 超时、配额

**取消**：
用户在投递前主动放弃整次口述；系统终止当前处理，清除会话内音频和文本，并且不投递、不写入本地历史。
_Avoid_: 失败、超时、达到上限

**切换式口述**：
用户完整单击一次全局快捷键开始口述，再完整单击一次结束口述的默认录音交互；取消使用独立命令。
_Avoid_: 按住说、松开结束

**原始转录**：语音识别器根据音频产生、尚未经过格式化和词典校正的文字。_Avoid_:
原文、识别稿

**最终转录**：语音识别完成后得到的稳定文字，是后续可选变换阶段的输入。_Avoid_:
精炼文本、AI 文案

**规则标准化**：在可选 LLM
精炼之后执行的确定性变换。当前只把已存在于个人词典中的 ASCII
字母数字标准词统一为用户保存的大小写和全角/半角形式，例如 `openai`、`OPENAI` 和
`ＯｐｅｎＡＩ` 统一为 `OpenAI`；不把 `open ai`
等不同词序列推断成同一词，也不使用错误写法做语义映射。_Avoid_: 精炼、润色

**精炼文本**： LLM
在不改变事实、观点和表达目的的前提下，对安全本地清理后的最终转录去除口语冗余、处理自我修正并整理结构后得到的文字。规则标准化在它之后运行。
_Avoid_: 回答、续写、AI 文案

**精炼模型**：被限制为只处理最终转录的 LLM
Provider；模型能力可以是通用的，但产品不向它开放问答、屏幕上下文或任务执行。
_Avoid_: 语音助手、聊天机器人

**投递**：
将精炼文本或降级后的最终转录写入实际投递时光标所在的位置，不包含自动发送消息。
_Avoid_: 发送、粘贴

**投递目标**：
实际投递发生时获得焦点的可编辑控件及其光标或选区；它不绑定口述开始时的应用或输入框。存在选区时最终文本替换选区，但
Saymore 不读取或处理选区原文。_Avoid_: 原始输入框、录音目标

**Secure 输入控件**：由操作系统标记为密码或其他安全文本输入的控件。Saymore
可以向实际投递时获得焦点的 Secure
输入控件尝试受限投递，但不读取或验证其中的内容，且该次口述不写入本地历史。
_Avoid_: 普通不可编辑控件、投递失败目标

**受限投递**：
向无法安全观察内容或选区的投递目标发起的一次不可验证投递；它只表示系统已发出输入操作，不等同于确认文字已经写入，也不因无法验证而自动判定为失败。
_Avoid_: 投递成功、验证投递

**最终投递**：
口述结束且所有已启用的变换阶段完成后，将一个最终纯文本结果一次性写入目标输入位置；同一口述会话最多发起一次，不因失败或不可验证结果自动重试。结果浮层和历史复制使用同一文本表示，不生成
HTML 或 RTF。_Avoid_: 实时投递、流式输入

**临时剪贴板投递**：macOS 直接 Accessibility
写入明确失败或目标控件不提供可验证选区范围时的降级路径。Saymore
只在单次操作内存中保存当前系统剪贴板、写入最终纯文本、触发粘贴，并在用户没有同时修改剪贴板时恢复原内容。该恢复快照在操作后立即释放，不进入本地历史；第三方剪贴板管理器自己的历史不受
Saymore 控制。_Avoid_: 本地历史、结果浮层复制、永久剪贴板记录

**切换式口述**：
完整单击一次全局触发器开始录音，再完整单击一次结束并处理；首次安装只有一个主触发器，macOS
当前默认为右
`Command`。主触发器可在输入框中直接录入替换，“添加另一个”用于添加并行的键盘或鼠标触发器。左右主按键和滚轮滚动不可绑定。Saymore
可录音时消费已绑定鼠标事件，不可录音时恢复透传。录音期间再次触发任一已绑定触发器都会结束同一会话。
_Avoid_: 按住模式、系统听写

**应用行为设置**：登录时启动和 Dock 图标显示两个独立开关，默认都开启。关闭 Dock
图标后菜单栏入口仍然存在；关闭设置或历史窗口不退出应用，只有菜单栏退出或 `Cmd+Q`
才终止进程。_Avoid_: 常驻状态、单一入口模式

**本地设置变更**：对 Saymore
保存在本机的产品偏好发起的一次原子变更；它只改变明确指定的偏好并保留其他偏好。Provider
配置和登录时启动等由操作系统持有的状态不属于本地设置变更。_Avoid_: Provider
配置变更、操作系统设置变更、界面暂存状态

**活动会话退出**：
录音中退出会取消口述并立即终止进程；处理阶段退出时由用户选择等待完成或立即丢弃。等待
30
秒后只提供继续等待或明确丢弃，不自动删除内容。等待完成后不直接投递：历史开启则保存结果后退出，历史关闭则在结果浮层中复制或明确放弃后退出。
_Avoid_: 关闭窗口、暂停 Saymore

**暂停 Saymore**：
保留应用进程、菜单栏和设置入口，但禁用所有录音触发器并恢复鼠标事件透传。录音中暂停等同取消当前口述。暂停状态跨应用重启和系统登录保留，只有用户从菜单栏明确恢复才重新启用触发器。
_Avoid_: 退出应用、暂停录音

**音频设置**：麦克风默认自动检测，即每次录音开始时使用 macOS
当前默认输入设备；也可由用户固定设备。录音过程中设备断开时立即结束，保留已采集音频继续识别，本次会话不切换设备。交互声音与“语音输入时静音其他活动音频”分别默认开启并可关闭。交互声音只用于录音开始和成功投递结束；主动取消、录音失败、识别失败和投递失败只使用视觉反馈。开始提示音播放后再静音；录音一结束就恢复音量，ASR
与精炼期间不继续静音。若用户在录音期间主动调节音量，则保留用户的新设置而不恢复旧值。
_Avoid_: ASR 音频参数、系统声音设置

**空会话**：VAD 没有检测到有效语音的录音。它正常结束，但不调用 ASR 或
LLM、不投递、不写历史、不显示结果浮层，只给出短暂的“未检测到语音”反馈。_Avoid_:
取消、识别失败

**渐进权限引导**：
首次使用时依次说明并申请麦克风和辅助功能权限，同时验证平台默认快捷键；只运行本地麦克风电平测试，不配置
ASR、不下载模型，也不执行第一次真实听写。缺失条件只禁用相关能力；权限拒绝后只提供系统设置入口和重新检测，不循环弹窗；不申请屏幕录制、通讯录或完整磁盘访问。
_Avoid_: 一次性权限墙、后台权限申请

**结果浮层**：
明确找不到投递目标或投递失败时展示只读最终文本并允许用户复制的恢复界面；仅仅无法验证受限投递不自动展示，避免文字已经写入后再次打断用户。结果浮层保持到复制或主动关闭，并在本地历史启用时于新口述开始前保存未处理结果。
_Avoid_: 预览浮层、错误弹窗

**仅结果浮层模式**：
辅助功能权限未授予或被撤销时的降级模式。录音、识别和精炼仍可运行，但不调用直接投递接口，最终文本始终进入结果浮层；授权恢复后下一次口述重新启用直接投递。
_Avoid_: 投递失败、剪贴板模式

**状态浮条**：
正常口述时显示录音、处理、成功或失败状态及音量反馈的轻量界面；它不展示转录文本，并可由用户关闭。
_Avoid_: 结果浮层、文本预览

**本地历史**：
保存在用户设备上的最终输出记录；它可以包含时间、投递状态和启用阶段，但不包含音频。普通发布只保存最终交付结果；调试构建或显式启用
`history-experiments` 的测试构建还可在同一加密记录中保存 ASR
原始结果和成功通过检查的 LLM
精炼结果，用于三阶段对比。首版只支持查看、复制、单条删除、全部清空和按期限自动清理，不承担搜索、标签、收藏或同步职责。
_Avoid_: 录音历史、调试日志

**个人词典**：用户在应用内手动添加、通过 CSV
导入，或由高可信用户纠正自动加入的正式词条集合。它记忆用户希望以后稳定复用的字面形式，包括姓名、品牌、项目名、代码标识符、行业术语和中文工作短语，不以公共词典是否收录为边界。
_Avoid_: 知识库、记忆、候选审核队列

**词汇观察**： Saymore
投递文本后，用户在同一控件和局部范围内对该文本所做的稳定纠正证据。它在口述会话完成后异步产生，不延长会话或改变已经确定的投递结果，也不包括模型差异或持久化完整输入框及整段上下文。
_Avoid_: 原始转录历史、完整上下文、待确认候选

**自动词典学习**：
根据词汇观察、候选置信度和重复证据，把高置信字面形式自动加入个人词典，并提供可理解、可撤销且可停止再次学习的反馈。它不从
ASR/LLM 输出差异猜测用户意图，也不创建错误写法到标准写法的强制映射。_Avoid_:
ASR/LLM 差异学习、全局字符串替换

**词典候选分类**：
对已经归因到用户局部纠正的候选做结构、词性、类型和复用价值判断，以区分稳定工作词汇与一次性句子改写。本地确定性规则先执行，单个
ASCII
字母因歧义过高而不参与自动学习；语义模型只复核未被本地拒绝的候选，不能覆盖本地拒绝、绕过本地归因和隐私门槛，也不能与转录精炼
Prompt 混用。_Avoid_: 真假单词判断、转录润色

**词典静默期**：
用户删除自动词条后，系统用于阻止同一字面形式再次被自动学习的内部状态。普通用户不需要管理复杂候选状态，但可以通过手动添加重新授权该词条。
_Avoid_: 禁用词条、永久屏蔽

**词典条目**：
正式的标准写法、可选语言标签、来源和时间。旧的常见错误写法字段为数据兼容和未来高可信学习保留，当前不由
UI 或 CSV 创建，也不参与本地语义映射。MVP
中词条全局生效并按语言隔离，不区分目标应用；当前规则标准化只统一同一 ASCII
字母数字词的大小写和全角/半角形式，并避开 URL、邮箱、路径、域名和下划线标识符。
_Avoid_: 词汇观察、知识条目

**词典文件**： CSV
是当前唯一的词典文件入口，用于从其他产品迁移正式词条；每行包含标准写法和可选语言标签。当前不提供
Markdown、JSON 或其他导出，也不把外部文件作为第二个数据源。_Avoid_:
完整应用备份、实时同步文件、Markdown 投影、导出

**本地数据密钥**：用于 AES-256-GCM 加密本地听写历史的随机 32-byte 密钥，保存在
macOS Keychain 或 Windows Credential Manager。它与 Provider API Key
分离；密钥不可用时停止访问旧历史，用户可显式删除旧历史并重新初始化，不得静默改用明文。个人词典和词汇观察不使用该密钥，也不受历史重置影响。
_Avoid_: API Key、用户密码

**Provider 数据确认**：云端 Provider
首次启用前的数据发送确认，明确发送内容、目标服务地址，以及是否包含本次相关的正式词典条目或用于候选分类的最小纠正片段。更换
Provider、Base URL 或数据范围后重新确认；本地 Provider
不需要，完整词汇观察、累计证据和输入框内容永远不发送。_Avoid_:
通用隐私政策确认、每次口述确认

**可选遥测**：
默认关闭、由用户分别主动开启的匿名使用统计和崩溃报告。事件采用字段白名单，禁止包含音频、任何文本内容、历史、词典、候选、Provider
地址、凭据或 Provider 请求响应。_Avoid_: 默认诊断、听写质量数据上传

**应用更新检查**：
默认开启的签名版本查询，不等于静默更新，也不是遥测。发现新版本后由用户确认，社区构建也可直接下载并安装经过
Saymore 更新签名验证的产物；请求不携带稳定设备标识或用户数据。_Avoid_:
静默更新、更新遥测

**Saymore 更新签名**：独立于 Apple Developer ID
的项目发布签名。客户端用内置公钥验证更新清单和安装包，签名无效时拒绝安装；私钥只存在于受保护的发布环境。它保证更新来自
Saymore，但不能替代 Gatekeeper 对 Apple 开发者身份的信任。_Avoid_: Apple
代码签名、文件哈希

**社区构建**：当前没有 Apple Developer Program 账号时发布的未公证 macOS
构建。它使用 Saymore 自有发布签名和哈希验证产物完整性，但需要用户按说明通过
Gatekeeper 手动放行，不能称为 Apple 签名或公证构建。_Avoid_: 正式 macOS
构建、已签名应用

**正式 macOS 构建**：取得 Apple Developer Program 账号后，使用 Developer ID
代码签名并通过 Apple Notarization、面向普通用户分发的安装包。_Avoid_:
社区构建、源码构建

**可信发布源**：MVP 中唯一发布真相是 GitHub Releases；官网只链接具体
Release。一个完整发布包含 DMG 安装包、应用内更新归档、更新清单、Saymore
更新签名、SHA-256 和发行说明。_Avoid_: 官网镜像、第三方软件源

**当前 ASR
Provider**：用户明确选择、下一次听写实际使用的一个本地模型或云端识别服务。同一时刻只存在一个当前
Provider；应用不根据语言、网络或性能静默替用户改选。_Avoid_:
推荐默认模型、自动模型路由、自动云端回退

**已安装本地模型**：已完成下载、哈希校验和原子激活，可在没有网络时加载的本地语音模型。设备可以同时保留多个已安装模型，但只有一个可以成为当前
ASR Provider；安装或切换模型不会自动删除其他模型。_Avoid_:
当前模型、下载缓存、自动清理模型

**模型下载任务**：把一个上游固定模型制品下载到临时文件、支持网络中断和应用重启后自动续传，并在完整哈希校验后原子激活的持久任务。上游不支持断点续传或制品身份变化时，任务明确提示并从头下载；未完成文件不是已安装模型。
_Avoid_: 内存下载、残缺模型、手动暂停

**模型下载队列**：本地语音模型下载任务的持久先进先出队列。同一时间只执行一个下载，其余任务显示排队状态；用户可以取消下载中或排队中的任务。
_Avoid_: 并行模型下载、隐藏后台下载

**取消模型下载**：用户主动终止下载中或排队中的模型任务。取消会删除队列记录和对应临时分片并释放磁盘；网络中断或应用退出不是取消，仍保留可验证的断点。
_Avoid_: 暂停下载、取消后保留缓存

**模型安装空间预检**：创建下载任务前，按剩余下载、临时文件、解压或最终安装副本及安全余量计算峰值磁盘需求。空间不足时不创建任务并显示空间缺口；下载途中耗尽空间时保留可验证断点，但不激活模型。
_Avoid_: 只检查模型标称大小、边下边覆盖、磁盘不足后激活

**本地设备能力评估**：应用只在设备本地读取操作系统、处理器架构、物理内存、必要的
CPU
指令集和可用推理加速后端，再与签名适配清单中每个模型经过目标设备实测的最低和推荐要求比较。已知硬不兼容或低于会导致加载失败、进程崩溃或明显系统风险的最低要求时禁止安装；达到最低要求但低于推荐要求时允许用户确认后安装并显示性能警告。该结果用于安装资格和风险提示，不承诺具体识别速度，也不默认上传硬件信息。
_Avoid_: 精确性能预测、按营销规格猜测、上传设备指纹

**首批本地语音模型**：模型页“语音模型”分类下首批适配的三个本地 ASR 模型，固定为
Paraformer、Whisper large-v3-turbo 和 Qwen3-ASR
1.7B。模型身份已经确定，但每个模型用于 macOS 与 Windows
的精确运行时制品仍需独立固定和验证。_Avoid_:
三个产品方向、文本润色模型、任意社区模型列表

**Saymore 验证模型**：第三方或自有模型中，被 Saymore
主动选型、适配和验证，再由可信适配清单固定上游官方版本、许可证、下载地址、哈希、兼容范围和测试结果的模型。默认模型权重必须允许个人与商业本地推理且不限制合法用途，运行时代码采用
OSI
批准许可证。默认模型不能要求账号注册、个人信息或额外条款接受。清单证明适配元数据经过
Saymore 验证，不表示 Saymore 托管、拥有或创作上游模型。_Avoid_: Saymore
自研基础模型、社区提交模型、任意模型执行器

**上游固定模型制品**：由模型项目或其维护者在官方仓库、官方模型站点或正式发布页提供，并由不可变版本或提交和文件哈希共同确定身份的模型文件集合。Saymore
直接从该官方来源下载，不重新打包或镜像；客户端只下载适配清单列出的数据文件，不从模型制品执行代码或安装脚本。
_Avoid_: Saymore 托管模型、第三方未知镜像、浮动最新版、在线模型市场

**签名适配清单**：由 Saymore
签名或随签名应用内置的模型兼容性与供应链元数据。客户端验证可信来源以及模型文件哈希和大小；验证失败时拒绝安装。清单指向上游官方固定制品，并保留上游来源、版本和许可证。
_Avoid_: 模型签名、模型版权证明

**模型切换保护**：
用户选择的新模型只有在完整性校验、加载和最小自检成功后才原子替换当前 ASR
Provider；失败时当前选择保持不变，并提供重试、修复和手动改选。应用不自动选择另一个本地模型或云端服务；模型删除始终需要用户确认。
_Avoid_: 跨模型自动回退、自动云端回退、模型自动清理

**删除当前模型**：用户确认删除当前本地模型后，模型文件被移除且当前 ASR Provider
变为空，真实听写保持禁用，直到用户手动选择另一个本地模型或云端服务。应用不会因其他模型已安装而自动改选。
_Avoid_: 禁止删除当前模型、删除后自动切换

**识别能力**：某个 ASR Provider
明确声明并经过验证的语言、自动检测、句内混说、地区变体、热词、流式和离线能力集合。
_Avoid_: 产品语言列表、模型宣传能力

**语言策略**：
一次口述采用的语言选择方式，可以是自动检测、带偏好的自动检测或固定语言；可选策略受当前识别能力约束。
_Avoid_: 界面语言、翻译模式

# Typeless 与闪电说的个人词典学习机制

调研日期：2026-07-11

## 结论

- **Typeless 已公开声称支持自动加入个人词汇。**其官网把 Personal Vocabulary 描述为可自动或手动加入词汇，并据此提高姓名、术语等内容的准确性。但官方没有公开自动加入的触发条件、是否先征求确认、候选词从哪里提取、保留多久以及如何逐条审计。
- **闪电说已验证的是手动“记忆”维护。**用户填写正确内容，还可以填写容易被识别错的内容，形成错误写法到正确写法的映射。没有找到官方资料证明它会从历史、用户编辑或纠错行为中自动新增词条。
- 两者都没有公开足够信息，支持我们照搬一套已验证的“自动学习算法”。Typeless 验证了产品能力的存在，但没有披露交互和实现；闪电说披露的手动映射更透明、可控。
- 因此 Saymore 应当支持个人词汇学习，但不应把未经确认的猜测静默写入正式词典。推荐组合两者优点：**自动发现候选词，用户确认后加入；同时保留手动的正确词及错误词映射。**

## 术语边界

| 机制 | 本报告中的定义 | 竞品证据 |
|---|---|---|
| 静默自动添加 | 系统无需用户逐条确认，直接把词条写入正式个人词典 | Typeless 只证明“可自动加入”，未证明是否静默 |
| 确认式建议 | 系统提出候选词，用户确认后才写入正式词典 | 两家均未找到明确的一手资料 |
| 手动添加 | 用户主动填写并保存词条 | Typeless 官网明确提到；闪电说文档明确说明 |
| 从用户编辑或纠错中学习 | 比较系统输出与用户之后的修改，据此产生或加入词条 | 两家均未找到明确的一手资料 |
| 从历史或上下文自动提取 | 扫描历史转录、屏幕或其他上下文，抽取专有名词 | 两家均未找到明确的一手资料 |
| 写作风格学习 | 学习语气、格式或表达习惯 | Typeless 有该功能，但它不是个人词典学习 |

这里必须区分“自动个人词汇”和“自动个性化写作风格”。Typeless macOS v0.9.0 的发布说明称个性化功能会自动学习用户的写作风格，并提供关闭开关；这不能证明它如何学习姓名或术语，也不能替代个人词典证据。[Typeless macOS 发布说明](https://www.typeless.com/help/release-notes/macos)

## Typeless

### 已验证事实

1. Typeless 官网的 Personal Vocabulary 功能明确表示，个人词汇可以自动或手动加入，用来持续提高姓名、术语及自定义词汇的准确性。[Typeless 官网](https://www.typeless.com/)
2. Typeless 的 macOS 发布说明另外描述了自动学习写作风格的 Personalization，并说明用户可在设置中关闭。这是风格个性化，不应被当作个人词典的实现说明。[Typeless macOS 发布说明](https://www.typeless.com/help/release-notes/macos)
3. Typeless 提供历史记录用于找回转录；其帮助文档将 History 描述为恢复遗漏文本的途径。这证明存在转录历史产品能力，但没有证明历史会被用于抽取词条。[找回遗漏转录](https://www.typeless.com/help/troubleshooting/missing-transcript)
4. Typeless 提供数据控制和隐私说明，但公开页面没有给出“自动词汇条目”的独立数据模型、保留期限或逐条审计方式。[Typeless Data Controls](https://www.typeless.com/data-controls)；[Typeless Privacy Policy](https://www.typeless.com/privacy)

### 未公开或无法验证

- “自动加入”是否为静默添加，还是先展示建议。
- 候选词是否来自用户修改、重复口述、通讯录、历史、屏幕上下文或其他信号。
- 用户能否查看自动词与手动词的来源差异。
- 自动词条的保留期限、同步范围和逐条删除方式。
- 词汇是在 ASR 热词层、规则层还是 LLM 精炼层使用。
- 自动学习算法、置信度阈值和回滚机制。

因此，能准确表达的结论是“Typeless 声称支持自动个人词汇”，不能进一步表述成“Typeless 会监控用户修改并静默学习”。后者目前没有一手证据。

## 闪电说

### 已验证事实

1. 闪电说把个人词汇能力称为“记忆”。官方文档要求用户填写“正确内容”，并允许额外填写“错误内容”，从而让容易识别错的形式映射到期望形式。[闪电说：记忆](https://shandianshuo.cn/docs/beginner/memory)
2. 这是一种显式、用户维护的词条机制：用户知道正式词条是什么，也能为同一正确形式补充常见误识别形式。[闪电说：记忆](https://shandianshuo.cn/docs/beginner/memory)
3. 闪电说同时支持不同语音识别模型和快速大模型；“记忆”不应与模型本身的训练或自动学习混为一谈。[闪电说：模型](https://shandianshuo.cn/docs/beginner/model)

### 未找到一手证据

- 自动从用户后续编辑或纠错中新增记忆。
- 自动从历史记录提取姓名、产品名或术语。
- 系统提出待确认的词条建议。
- 无需用户确认，静默新增正式记忆。
- 自动学习算法或模型训练细节。

在没有官方证据时，应把闪电说归类为**手动词典及错误到正确映射**，而不是自动个人词典。

| 维度 | Typeless | 闪电说 |

| 手动加入 | 已明确宣传 | 已明确文档化 |
| 自动加入 | 已明确宣传存在 | 未验证 |
| 确认式建议 | 未公开 | 未验证 |
| 错误词到正确词映射 | 未公开 | 已明确文档化 |

| 从历史/上下文提取 | 未公开 | 未验证 |
| 用户关闭自动能力 | 写作风格学习有开关；词汇自动加入的独立开关未验证 | 不适用，未验证自动能力 |
| 实现细节 | 未披露 | 只披露用户配置方式，未披露内部管线 |

## Saymore 建议

按照“竞品有则做，没有则不做”的产品原则，Saymore 应做个人词汇学习，因为 Typeless 已明确提供自动加入能力；但不能声称复刻 Typeless 的具体机制，因为该机制没有公开。

推荐采用以下可解释方案：

1. **正式词典继续支持手动维护。**每条包含“正确形式”和零个或多个“常见错误形式”，采用闪电说公开方案中最有价值的部分。
2. **增加自动候选，而不是静默正式入库。**系统发现高置信度专有词后放入候选列表；只有用户确认才进入正式词典。这保留 Typeless 的低操作成本，同时避免一次误识别被长期固化。
3. **候选发现默认开启，但可关闭。**关闭后只保留手动词典。候选条目应显示候选词、来源类型、发现时间和建议原因。
4. **MVP 不从目标应用中的后续编辑学习。**Saymore 将最终文本写入任意应用后，可靠追踪用户如何修改它需要额外的可访问性监控，隐私与正确性成本很高；两家也没有一手资料证明采用了该机制。
5. **MVP 不扫描屏幕、联系人或外部知识库。**这与当前“只做语音识别输入”的边界一致，也没有竞品个人词汇资料要求这样做。
6. **候选仅基于 Saymore 已获准处理的数据。**优先使用本次 ASR 候选、用户主动修改 Saymore 结果浮层的内容（若未来允许编辑）以及重复出现的未登录词。不要为了学习而额外保存音频或完整中间响应。
7. **词典和候选均可审计、删除和导出。**用户应能逐条删除、清空候选、关闭发现功能；删除正式词条后，不应立即由相同证据自动重新添加。

建议的数据状态明确分离：

```text
ObservedTerm -> SuggestedTerm -> ConfirmedDictionaryEntry
                      |                    |
                    Reject              Disable/Delete
```

`SuggestedTerm` 不是正式词典条目，不能直接影响输出。拒绝应形成短期抑制记录，避免同一错误建议反复打扰；是否长期保留抑制记录需要另行决定。

## 建议决策

Saymore 不采用“完全不做自动学习”，也不采用“未经确认静默入库”。采用：

> **手动正确词/错误词映射 + 自动候选 + 用户确认入库。**

这不是声称 Typeless 或闪电说已经公开使用同一交互，而是在已验证能力边界内，组合 Typeless 的自动发现价值与闪电说的显式可控词条结构。

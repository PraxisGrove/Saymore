# Saymore 本地 ASR Beta 发布决策地图

状态：当前有效，等待三个模型的精确运行时制品决策 日期：2026-07-24

## 目标与边界

尽快让普通用户完成“下载模型 -> 选择本地识别 -> 离线听写 ->
失败可恢复”的完整闭环。首版直接使用上游社区的官方固定制品，不建设 Saymore OSS
或镜像，不训练、微调或自行量化。macOS 与 Windows
同步发布。模型页“语音模型”分类固定适配 Paraformer、Whisper large-v3-turbo 和
Qwen3-ASR 1.7B 三个本地 ASR 模型；它们不是三个产品方向，也不与文本润色模型混排。

当前边界是 `ASR-020`。三个模型身份和“语音模型”页面归属已经确定，但 Whisper 官方
PyTorch checkpoint、Qwen 官方 BF16
和社区运行时转换并不自动等于双端桌面可用制品。每个模型必须先固定实际运行时格式与上游不可变版本，才能实现对应
Adapter 和下载清单。

## Beta 必须具备

- 在独立模型页的“语音模型”分类展示 Paraformer、Whisper large-v3-turbo 和
  Qwen3-ASR 1.7B，并明确大小、来源、许可证、适用场景和“音频不离开设备”。
- 三个本地模型平级展示，不默认推荐、不预选、不自动下载；未来 Saymore 自有云端
  ASR 也由用户主动选择，不自动取代当前 Provider。
- 允许多个本地模型同时安装，但只有一个当前
  Provider；模型页显示单模型和全部本地模型的磁盘占用。
- 当前模型可以在二次确认后删除；删除后当前 Provider
  置空、听写禁用，不自动选择其他本地模型或云端服务。
- 三个模型都必须在 macOS 与 Windows
  完成安装、加载和真实听写闭环；第一版不使用“即将推出”占位，也不以单个模型完成代替三模型交付。
- 可信模型元数据随签名应用内置，不为首版建设远程模型注册表。
- 模型只从上游官方固定版本或提交下载，不使用浮动最新版地址，也不通过 Saymore
  自建 OSS 或镜像分发。
- 下载失败可重试，哈希不匹配和残缺文件绝不激活。
- 下载在网络恢复或应用重启后自动断点续传；首版不要求手动暂停，上游不支持续传或制品变化时明确提示并从头下载。
- 多个下载请求进入持久先进先出队列，同一时间只下载一个模型；其余显示排队状态，用户可以取消。
- 主动取消下载会删除队列记录和全部临时分片；网络中断或应用退出仍保留断点。
- 下载前检查峰值磁盘需求并显示缺口；空间不足不创建任务，途中耗尽保留断点但不激活模型。
- 设备能力检测只在本地读取操作系统、架构、物理内存、必要 CPU
  指令集和可用推理加速后端，并与每个模型经过目标设备实测的最低和推荐要求比较；检测信息不默认上传。
- 已知硬不兼容或低于会导致加载失败、进程崩溃或明显系统风险的最低要求时禁用安装；达到最低要求但低于推荐要求时允许用户确认后安装并明确警告。
- 模型加载失败、录音取消、无有效语音和识别失败都有明确状态，不产生错误投递。
- 新选择通过校验、加载和最小自检后才原子生效；失败时不跨本地模型或云端自动回退。
- 本地识别不依赖网络；云端 ASR 保留为用户主动选择的替代 Provider。
- macOS 与 Windows 使用相同模型身份和能力声明，并同步通过发布门槛。

## Beta 后再做

- 模型训练、微调或自行量化。
- Saymore 自建 OSS 或镜像、远程签名适配清单、手动暂停和手动离线导入。
- 完整 VAD/端点检测优化、公开 benchmark 服务、模型目录、社区和排行榜。
- 为所有语言和硬件档位作统一承诺；首版只声明实际通过测试的能力。

## 已知现状

- 应用层已经定义流式识别会话契约，生产桌面运行时目前只创建云端 ASR Adapter。
- Paraformer Q8 已有本地 Rust/sherpa-onnx 探针和性能基准，但尚不是生产 Adapter。
- 当前固定探针样本的最终文本仍有结尾重复和截断，必须先修复，不能用模型微调掩盖会话结束错误。
- 已安装模型存储目前只保存元数据，尚不负责下载、校验、原子激活、删除或损坏恢复。
- 当前模型页是云端 Provider 配置界面，尚未提供本地模型浏览、比较或生命周期管理。

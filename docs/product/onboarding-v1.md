# Saymore Onboarding V1

## 目标

让首次安装用户在最短路径内完成三件事：确认麦克风可用、学会语音快捷键、理解并开启辅助功能权限。首版不要求账号，不要求配置
ASR API，也不发起语音识别或大语言模型请求。

Onboarding
只在当前用户第一次安装并启动时自动出现。完成状态保存在本机；升级版本不重复弹出。设置页保留“重新运行新手引导”入口，删除
Saymore 的全部本地数据后会重新出现。

## 页面结构

### 1. 欢迎

标题：`用说话代替打字`

正文：`Saymore 让你在任何输入框中用语音快速输入。先花一分钟完成麦克风、快捷键和辅助功能设置。`

主按钮：`开始设置`

画面只展示 Saymore
品牌、一个简短的语音波形动效和主按钮，不放功能列表、账号入口或模型配置。

### 2. 麦克风与快捷键

标题：`测试你的麦克风`

页面分成两个连续区域，但仍属于同一步骤。

麦克风区域：

- 未授权时显示用途说明：`Saymore 需要麦克风来听取你的语音。音频测试只在本机进行。`
- 主操作为 `允许麦克风`，点击后触发 macOS 系统授权窗口。
- 授权后显示当前输入设备和实时音量条。
- 有声音输入时显示 `麦克风工作正常`；持续无输入时提供设备选择和 `重新检测`。

快捷键区域：

- 引导页只展示平台默认快捷键，不提供编辑、删除或“添加另一个”：macOS 为
  `Right Command`，Windows 为 `Right Alt`。
- 提示：`按下默认快捷键，试着说几句话。看到波形随声音变化，就说明麦克风工作正常。再次按下即可结束测试。`
- 用户按下平台默认快捷键后，只启动本地麦克风电平测试，不调用 ASR，不保存音频。
- 首次触发成功后，状态变为 `快捷键正常`。
- 多组快捷键的添加、删除、冲突提示和自定义录入只属于设置页，不出现在 Onboarding
  中。

只有“麦克风已授权且检测到输入”和“快捷键至少成功触发一次”同时成立时，`继续`按钮进入完成态。应提供
`暂时跳过`，避免硬件或系统权限问题让用户困在引导中。

### 3. 辅助功能

标题：`让 Saymore 在其他应用中输入文字`

正文：`辅助功能权限用于监听全局快捷键，并把识别结果写入你当前使用的输入框。Saymore 不会借此读取密码框内容。`

主按钮：`打开系统设置`

页面持续检测系统授权状态：

- 未开启：展示分步截图位置和 `重新检测`。
- 已开启：自动显示成功状态，但不强制立刻跳页，避免用户看不到反馈。
- 被拒绝或系统限制：保留说明和再次打开系统设置的入口。
- 从系统设置回到 Saymore
  时立即重新读取权限，同时保留短周期轮询作为兜底；首页、设置页和首次引导共享同一系统状态，不缓存授权结果。

首版不申请屏幕录制权限，因为当前核心语音输入不需要读取屏幕。

若用户暂时跳过引导，macOS
在辅助功能尚未开启时仍会以低开销方式检测已配置快捷键。用户按下快捷键时，不开始录音，而是显示
Saymore
自己的权限提示浮窗；浮窗只提供关闭和直接打开“辅助功能”系统设置的入口。Windows
不使用该降级路径。设置页、引导页和该浮窗打开辅助功能设置时，不再同时触发 macOS
自带的二次询问弹窗。

### 4. 完成

标题：`Saymore 已准备好`

正文：`在任意输入框中按下你的快捷键开始说话，再按一次完成。`

页面显示平台默认快捷键：macOS 为 `Right Command`，Windows 为
`Right Alt`。用户完成引导后可在设置页添加或更换快捷键。

补充说明：`Saymore 会在后台保持可用。你可以从屏幕顶部的 Saymore 图标打开、暂停或退出。`

提供 `登录时启动 Saymore` 开关，默认关闭，由用户明确选择。主按钮为 `开始使用`。

## 状态与持久化

- `not_started`：首次安装，启动时显示欢迎页。
- `in_progress`：记录当前步骤；应用意外退出后回到未完成步骤。
- `completed`：以后直接打开主窗口，不再自动弹出。
- `skipped`：视为已完成，但设置页可以重新进入。

权限状态不直接持久化为真值，每次显示页面时都读取 macOS
当前授权，避免用户之后在系统设置中撤销权限而界面仍显示成功。

## 视觉约束

- 使用独立的固定尺寸设置窗口，不在主窗口上叠多层卡片。
- 顶部显示 `1 / 4` 到 `4 / 4` 的简洁进度，不使用可点击步骤导航。
- 每页一个主操作，次要操作使用文本按钮。
- 麦克风电平条占据页面主要视觉区域，反馈必须实时且稳定，不能用预设动画假装收到声音。
- 权限系统弹窗出现前，先在 Saymore 页面解释用途。
- 麦克风与快捷键区域使用紧凑、对齐的工具面板，避免大面积灰底和无意义留白。
- 权限状态使用标题旁的小型非交互标签；打开设置、重新检测等操作必须保持明确的按钮外观。

## 实现边界

Onboarding
的麦克风测试应建立独立的本地测试会话，只读取瞬时电平并在结束时丢弃缓冲区。它不能复用当前需要
ASR 配置的完整听写完成流程，否则未配置 API 的新用户会在首次体验中看到无关错误。

Onboarding
只调用默认快捷键的监听能力，不包含快捷键管理。自定义录入、冲突规则、持久化和运行时更新统一由设置页的快捷键业务模块负责。

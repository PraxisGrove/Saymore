# 采用 PolyForm Shield 源码可用许可

状态：已接受

日期：2026-07-17

## 背景

Saymore
希望公开源码、允许个人和组织内部使用，并接受社区改进，同时保留语音输入产品和服务的商业化空间。原有
`MIT OR Apache-2.0` 许可允许第三方直接创建和商业化竞争产品，与该目标不一致。

## 决策

从本决策起，Saymore 官方源码采用未经修改的 PolyForm Shield License
1.0.0，并在根目录 `LICENSE` 中提供完整条款和 Required Notice。

该许可允许个人使用、组织内部使用、学习、修改、分发和其他非竞争性用途。任何产品或服务只要与
Saymore 或许可方基于 Saymore
提供的产品形成竞争，就不属于许可用途；相关方必须另行取得商业许可。

Saymore 对外使用“source-available”或“源码可用”描述，不将该许可宣传为 OSI
批准的开源许可。第三方依赖、字体、图标、音效和模型继续遵循各自许可证。

`LICENSE` 是项目许可条款的唯一事实来源。README
和产品文档只解释许可边界，不增加或修改法律条款。

## 贡献

Issue、设计讨论和不包含代码的反馈继续开放。在明确的 Contributor License
Agreement（CLA）签署流程上线前，不合并外部代码贡献。未来 CLA
必须让贡献者保留其版权，同时向 Saymore
的权利主体授予使用、修改、商业化和再许可贡献所需的明确权利及必要专利许可。

## 结果

- 用户可以审查源码并在许可范围内自托管或自行构建。
- 组织可以内部采用 Saymore，而无需仅因内部使用购买商业许可。
- 竞争产品、竞争服务和其他不在公开许可范围内的用途需要商业许可。
- 项目采用标准许可文本，不创建“Modified Apache”等自定义许可证。
- 商标、官方签名构建、更新服务、企业支持和商业许可由独立政策或合同管理。
